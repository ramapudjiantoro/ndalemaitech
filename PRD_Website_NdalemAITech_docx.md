# 1. Executive Summary

Dokumen ini adalah Product Requirements Document (PRD) untuk website resmi Ndalem AI Tech — sebuah bisnis teknologi hospitality yang dibangun oleh Rama Pudjiantoro sebagai solo founder.

Website ini bukan sekedar company profile. Website ini adalah instrumen sales dan credibility utama Ndalem AI Tech yang berfungsi untuk menarik, meyakinkan, dan mengkonversi calon client — yaitu owner guesthouse, homestay, dan penginapan kecil di Indonesia.

|  |
| --- |
| **Pernyataan Inti Produk**  Website Ndalem AI Tech dirancang untuk menjawab satu pertanyaan kunci dari calon client:  "Apakah saya bisa mempercayai orang ini untuk membangun sistem digital untuk bisnis penginapan saya?"  Setiap elemen website harus menjawab pertanyaan ini dengan YA — melalui bukti nyata, bukan klaim kosong. |

## 1.1 Tujuan Utama Website

Website Ndalem AI Tech memiliki tiga tujuan bisnis yang saling mendukung:

* Lead Generation — Mendatangkan calon client (guesthouse owner) yang tertarik membangun website dan booking system mandiri
* Credibility Building — Membuktikan bahwa Ndalem AI Tech bukan developer generik, melainkan hospitality technology specialist yang memahami bisnis penginapan dari dalam
* Sales Conversion — Mengkonversi visitor menjadi inquiry melalui CTA yang clear dan messaging yang relevan

## 1.2 Ringkasan Bisnis Ndalem AI Tech

Ndalem AI Tech adalah bisnis teknologi yang membantu owner guesthouse dan homestay membangun website dan sistem booking mandiri. Keunggulan utama bukan kemampuan coding — melainkan pengalaman founder sebagai operator bisnis hospitality nyata melalui Ndalem Pleret Guest House & Residences.

Saat ini bisnis berada pada tahap validation stage: sudah memiliki produk yang berjalan (deployed di Ndalem Pleret), belum memiliki paying client eksternal, dan sedang menjalani proses customer discovery aktif.

# 2. Goals & Success Metrics

## 2.1 Business Goals

Website ini harus secara langsung mendukung target bisnis jangka pendek Ndalem AI Tech:

| **Prioritas** | **Business Goal** | **Bagaimana Website Mendukung** |

| P0 — Kritis | Mendapatkan paying client pertama | Menghasilkan inquiry dari guesthouse owner yang qualified |
| P0 — Kritis | Memvalidasi market demand | Mengukur berapa banyak guesthouse owner yang datang organik dan tertarik |
| P1 — Penting | Membangun credibility sebagai specialist | Menampilkan case study Ndalem Pleret sebagai social proof nyata |

Keberhasilan website diukur melalui metrik berikut, dibagi berdasarkan fase waktu:

# 3. Target Audience & User Personas

## 3.1 Primary Audience

Target utama website adalah owner guesthouse, homestay, dan penginapan keluarga skala kecil hingga menengah di Indonesia. Bukan HRD, bukan manajer — langsung owner atau decision maker bisnis.

|  |
| --- |
| **Karakteristik Primary Audience**  Usia: 30–55 tahun  Peran: Owner atau co-owner bisnis penginapan (bukan manajer profesional)  Skala bisnis: 3–20 kamar  Literasi digital: Menengah — bisa pakai smartphone dan WhatsApp, tidak harus tech-savvy  Platform yang digunakan: Traveloka, Tiket.com, Booking.com, atau Instagram  Masalah yang mungkin dirasakan: Komisi OTA terlalu besar, tidak punya website sendiri, booking masih manual via WhatsApp  Motivasi utama: Ingin lebih mandiri secara digital tanpa bergantung penuh pada OTA |

## 3.2 User Personas

### Persona A — Bu Dewi, Owner Guesthouse Jogja

|  |  |
| --- | --- |
| **Profil** | Wanita, 42 tahun, owner guesthouse 8 kamar di Yogyakarta. Bisnis diwarisi dari orang tua. |
| **Aktivitas digital** | Aktif di Instagram dan WhatsApp. Jarang googling untuk solusi bisnis. |
| **Masalah utama** | Hampir 90% booking datang dari Traveloka. Komisi besar, merasa tidak punya kontrol atas data tamu. |
| **Keinginan** | Punya website sendiri yang bisa dilihat tamu, tapi tidak mau ribet urus teknologi. |
| **Kekhawatiran** | Takut mahal, takut susah dipakai, takut tidak ada yang bantu kalau ada masalah. |
| **Trigger untuk beli** | Melihat contoh nyata (Ndalem Pleret) dan tahu harga + waktu pengerjaan yang jelas. |

### Persona B — Pak Arif, Owner Homestay Bali

|  |  |
| --- | --- |
| **Profil** | Pria, 38 tahun, owner homestay 12 kamar di Seminyak. Buka bisnis sendiri 5 tahun lalu. |
| **Aktivitas digital** | Cukup tech-savvy. Pernah coba bikin website sendiri tapi tidak jalan. Pakai Booking.com dan Airbnb. |
| **Masalah utama** | Direct booking sangat sedikit. Tamu yang sudah pernah menginap tetap booking lewat OTA. Kehilangan repeat revenue. |
| **Keinginan** | Booking system yang bisa diakses tamu langsung, terintegrasi WhatsApp konfirmasi. |
| **Kekhawatiran** | Pernah kena tipu developer freelance, tidak deliver tepat waktu. Butuh jaminan. |
| **Trigger untuk beli** | Portofolio yang menunjukkan sistem sudah running + founder yang paham bisnis hospitality. |

### Persona C — Pak Heru / Bu Lastri, Owner Penginapan “Apa Adanya”

Persona gaptek yang merepresentasikan mayoritas pasar. Tidak aktif mencari solusi digital — penting dipahami untuk messaging dan content strategy.

|  |  |
| --- | --- |
| **Profil** | Bapak/Ibu, 50-65 tahun, owner penginapan 3-8 kamar. Bisnis turun-temurun atau dibuka tanpa rencana besar. |
| **Aktivitas digital** | Posting foto kamar di Facebook, kadang di grup jual-beli lokal. Tidak punya akun Instagram bisnis. Tidak kenal Google My Business. |
| **Mindset bisnis** | Ada tamu alhamdulillah, tidak ada ya alhamdulillah. Tidak merasa punya masalah digital karena tidak tahu apa yang selama ini hilang. |
| **Keinginan** | Kalau ada yang bantu simpel, murah, dan tidak perlu mengurus teknologi sendiri, mungkin mau coba. |
| **Kekhawatiran** | Takut mahal. Takut tidak bisa pakai. Takut tidak ada yang bisa dihubungi kalau ada masalah. |
| **Implikasi messaging** | Persona ini butuh edukasi dulu sebelum bisa convert. Pesan website harus relevan tanpa mengorbankan konversi Persona A & B. Konten blog (V1.1) lebih efektif untuk menjangkau segmen ini. |

# 4. Problem Statement

## 4.1 Masalah yang Dialami Calon Client

Ndalem AI Tech meyakini bahwa mayoritas owner guesthouse dan homestay menghadapi kombinasi dari masalah-masalah berikut. Catatan: sebagian masalah ini masih dalam tahap validasi dan belum sepenuhnya dikonfirmasi oleh market.

| **Masalah** | **Dampak Bisnis** | **Status Validasi** | **Prioritas** |
| --- | --- | --- | --- |
| Ketergantungan pada OTA (Traveloka, Booking.com) | Komisi 15–25% per booking, kehilangan data tamu, tidak ada repeat marketing | Asumsi kuat | P0 |
| Tidak punya website sendiri | Tidak ada kehadiran digital mandiri, credibility rendah di Google | Fakta umum | P0 |
| Proses booking manual via WhatsApp | Error rentan, double booking, tidak ada catatan terstruktur | Fakta umum | P0 |
| Tidak ada sistem manajemen tamu | Data tamu hilang setelah check-out, tidak bisa follow-up repeat booking | Hipotesis | P1 |
| Tidak tahu cara atasi masalah digital | Ragu-ragu, tidak tahu mulai dari mana, takut biaya besar | Asumsi | P1 |

## 4.2 Masalah Ndalem AI Tech dalam Menjual Layanannya

Selain masalah client, Ndalem AI Tech juga memiliki tantangan internal yang harus diatasi oleh website ini:

|  |
| --- |
| **Tantangan Sales Ndalem AI Tech**  1. Founder belum dikenal — tidak ada brand awareness, tidak ada referral organik yang kuat  2. Sulit membedakan diri dari developer generik — tanpa bukti nyata, semua developer terlihat sama  3. Calon client skeptis — pernah kena tipu atau kecewa dengan developer sebelumnya  4. Sales masih one-on-one — belum ada aset digital yang bisa 'menjual' saat founder tidak ada  5. Portfolio masih satu (Ndalem Pleret) — butuh ditampilkan maksimal sebagai social proof |

## 4.3 Solusi yang Ditawarkan Website

Website Ndalem AI Tech harus menjadi solusi atas kedua kelompok masalah di atas secara bersamaan:

* Menyampaikan value proposition dengan bahasa yang dipahami guesthouse owner (bukan bahasa developer)
* Menampilkan bukti nyata (Ndalem Pleret) sebagai solusi konkret yang sudah terbukti berjalan
* Membedakan Ndalem AI Tech dari developer generik melalui messaging yang menonjolkan pengalaman hospitality
* Menyederhanakan proses kontak — satu klik ke WhatsApp atau form inquiry
* Membangun kepercayaan melalui transparansi tentang siapa founder dan bagaimana ia bekerja

## 7.2 User Flow Utama

Terdapat tiga jalur pengguna yang paling penting dan harus difasilitasi dengan mulus oleh website:

### Flow 1 — Discovery to Contact (Most Common)

* User menemukan website via Google / media sosial
* Mendarat di Homepage → membaca hero section
* Scroll down → melihat problem-solution → memahami relevansi
* Melihat portfolio Ndalem Pleret → mulai percaya
* Klik CTA → diarahkan ke Halaman Kontak atau WhatsApp langsung

### Flow 2 — Referral to Portfolio (Penting untuk Credibility)

* User direferensikan oleh kenalan
* Langsung mencari 'bukti nyata' → ke halaman Portfolio
* Membaca case study → melihat screenshot / demo
* Membaca tentang founder di halaman About
* Hubungi via WhatsApp atau form

### Flow 3 — Service Research (Calon Client yang Sudah Siap)

* User sudah tahu butuh website / booking system
* Langsung ke halaman Layanan → membaca paket
* Cek FAQ untuk pertanyaan teknis / harga / waktu
* Klik 'Mulai Sekarang' → ke halaman Kontak

# 8. Detailed Page Requirements

## 8.1 Homepage (/)

Homepage adalah halaman paling penting. Ini adalah titik masuk utama dan harus mampu meyakinkan visitor dalam 5 detik pertama bahwa Ndalem AI Tech relevan untuk masalah mereka.

### 8.1.1 Hero Section

|  |  |
| --- | --- |
| **Tujuan** | Menangkap perhatian, menyampaikan value proposition utama dalam satu kalimat, mendorong scroll atau klik CTA |
| **Headline Utama** | "Bangun Website dan Sistem Booking Penginapan Milikmu Sendiri" |
| **Sub-headline** | "Kami membantu owner guesthouse dan homestay punya aset digital mandiri — tidak bergantung penuh pada OTA." |
| **CTA Utama (Primary)** | "Konsultasi Gratis" → arahkan ke WhatsApp atau halaman Kontak |
| **CTA Sekunder** | "Lihat Hasil Kerja Kami" → arahkan ke halaman Portfolio |
| **Visual** | Background foto atau ilustrasi yang merepresentasikan guesthouse / hospitality Indonesia. Bukan stock photo generik. |
| **Social Proof Micro** | Kalimat singkat di bawah CTA: "Sudah digunakan oleh Ndalem Pleret Guest House & Residences" |

### 8.1.2 Problem Section — 'Apakah Ini Masalahmu?'

|  |  |
| --- | --- |
| **Tujuan** | Membangun empati dan resonansi dengan calon client. Membuat mereka merasa 'ini persis masalahku'. |
| **Judul Section** | "Apakah kamu menghadapi salah satu dari ini?" |
| **Konten** | 3–4 pain point dalam format kartu atau ikon: (1) Booking masih via WhatsApp manual, (2) Hampir semua tamu datang dari OTA, (3) Belum punya website sendiri, (4) Tidak tahu harus mulai dari mana |
| **Catatan Desain** | Gunakan bahasa langsung ke pembaca (kata 'kamu'). Hindari bahasa formal atau korporat. |

### 8.1.3 Solution Section — 'Apa yang Kami Tawarkan'

|  |  |
| --- | --- |
| **Tujuan** | Memperkenalkan solusi dengan cara yang mudah dipahami non-tech audience |
| **Judul Section** | "Yang Kami Bangun Untukmu" |
| **Item 1** | Website Penginapan — Tampilkan kamar, fasilitas, lokasi, dan harga. Profil digital milikmu sendiri. |
| **Item 2** | Sistem Booking Mandiri — Tamu bisa booking langsung, data tersimpan, konfirmasi otomatis via WhatsApp. |
| **Differentiator** | Box/highlight khusus: 'Kami bukan developer biasa. Kami juga operator guesthouse — jadi kami tahu sistem seperti apa yang benar-benar berguna.' |

### 8.1.4 Social Proof Section — 'Lihat Hasilnya'

|  |  |
| --- | --- |
| **Tujuan** | Membangun kepercayaan dengan menampilkan bukti konkret bahwa sistem sudah berjalan nyata |
| **Judul Section** | "Sudah Diterapkan Secara Nyata" |
| **Konten** | Preview singkat case study Ndalem Pleret: gambar website, 2–3 highlight fitur, quote singkat tentang manfaatnya |
| **CTA** | "Lihat Case Study Lengkap" → link ke halaman Portfolio |

# 11. Design & UX Guidelines

## 11.1 Design Principles

* Clarity over cleverness — Desain yang jelas lebih penting dari desain yang 'keren'
* Trust-first — Setiap elemen desain harus mendukung pembangunan kepercayaan
* Mobile-first — 70%+ traffic diprediksi dari smartphone; desain dimulai dari layar kecil
* Conversion-focused — Setiap halaman memiliki tujuan konversi yang jelas dengan CTA yang obvious
* Performance-conscious — Desain yang ringan dan cepat lebih baik dari desain yang berat dan lambat

## 11.4 UI Component Standards

### Button States

* Primary CTA: solid background, rounded corners, hover state dengan slight darkening
* Secondary CTA: outline button, hover fill
* Disabled state: 50% opacity, no pointer cursor
* Loading state: spinner animation saat form sedang disubmit

### Card Component

* Padding internal: 24–32px
* Border radius: 8–12px
* Box shadow subtle untuk depth: 0 2px 8px rgba(0,0,0,0.08)
* Hover state: slight elevation increase untuk interactive card

### Form Elements

* Input field border: 1px solid #CBD5E0
* Focus state: 2px outline #2E75B6
* Error state: border merah, error message merah di bawah field
* Placeholder text: tidak bisa dijadikan satu-satunya instruksi — selalu ada label

# 15. Risks, Constraints & Assumptions

## 15.1 Risiko dan Mitigasi

| **Risiko** | **Probabilitas** | **Dampak** | **Mitigasi** |
| --- | --- | --- | --- |
| Konten (foto, copy) terlambat disiapkan | Tinggi | Tinggi — blok development | Jadwalkan content gathering di minggu pertama sebelum coding dimulai |
| Scope creep — fitur terus bertambah | Sedang | Sedang — memperlambat launch | Disiplin dengan scope V1.0 yang sudah ditetapkan. Fitur baru masuk backlog V1.1 |
| Website tidak generate inquiry sama sekali | Sedang | Tinggi — kegagalan bisnis tujuan | A/B test messaging, ubah CTA, interview 3–5 guesthouse owner tentang hambatan mereka |
| Performance buruk di mobile karena aset besar | Sedang | Tinggi — UX rusak untuk target market | Wajib optimize semua gambar ke WebP < 200KB, lazy loading, next/image |
| Solo operator — tidak ada review kedua sebelum launch | Tinggi | Sedang — bug atau typo lolos | Minta minimal 2 orang non-developer review website sebelum go-live |

* Budget terbatas — tidak ada anggaran untuk designer profesional atau copywriter eksternal di tahap ini

* Brand identity (logo, palette) mungkin masih perlu finalisasi

* Nomor WhatsApp bisnis yang dedicated sudah tersedia
